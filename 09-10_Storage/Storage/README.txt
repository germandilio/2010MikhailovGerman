README - Storage (10 PeerGrade)

В приложении реализован весь стандартный функционал, а также 10 доп пункт.
Приложение написано на основе старого пиргрейда. Однако некоторые функции были удалены для облегчения проверки, а именно: открытие и сохранения склада происходит автоматически, количество товаров на складе не уменьшается при покупке пользователями(чтобы не добавлять их постоянно). В качестве шаблона загружен склад с машинами.

Взаимодействие со списком заказов/товаров/клиентов:
    - добавление товара в корзину происходит в форме при нажатии левой кнопкой мыши по товару. (два раза кликнуть на nodeTree чтобы посмотреть товары в категории).
    - при нажатии левой кнопкой мыши на заказ его можно оплатить (при наличии статуса Processed)
    - при нажатии левой кнопкй мыши на клиента можно просмотреть все его заказы (при нажатии на каждый заказ ему можно поменять статус)
   - правой кнопкой мыши по товару (удаление товара).

В режиме клиента большинство кнопок редактирования недосутпно. Аккаунт продавца фиксирован для облегчения проверки: (login: {Admin} password: {Admin001} ) Данные уже введены в форму и сохранены в базе данных.
При регистрации клиенат необходимо следовать подсказкам (красные кружки при неккоректных значениях).

Краткое инструкция:

Разделы классификатора:
Все взаимодейтсвие с разделами классификатора происходит с помощью контекстного меню (Парвая кнопка мыши). Двойной клик открывает товары в данном разделе.

Список товаров:
Клик левой кнопкой мыши открывает карточку товара.
Клик правой кнопкой мыши по товару вызывает удаление товара (требуется подтверждение)

Сохранение и открытие склада:
   Требуется ввести название папки(будет создана новая), где будут располагаться два ключевых файла.
   При открытии склада следует ввести имя папки в которой лежат те самые два файла.
(Примечание) Папки со складами нахоятся в SavedStorages (в корневой папке проекта).

В программе используюся nuget пакеты: СsvHelper , NewtonJson (должны установиться автоматически при первом запуске).
